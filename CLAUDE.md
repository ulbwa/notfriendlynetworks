# notfriendlynetworks — инструкции проекта

Репозиторий — наборы правил (sing-box rule-sets) по сервисам, ASN-провайдерам и странам.

## Структура

- `services/<name>/` — сервис (домены, подсети, Android-пакеты).
- `asn/<name>/` — IP-подсети хостинг/CDN-провайдера или ASN.
- `countries/<code>/` — подсети по странам (ISO-код).

В каждой папке:
- `ruleset.json` — sing-box rule-set (`version: 2`). **Обязателен.**
- `config.json` — конфиг папки (доступность по регионам + проба). **Опционален:** существует только если есть ключ `probe` или непустой ключ `availability`; иначе файла быть не должно (см. ниже).
- `README.md` — описание + статистика + доступность (единый шаблон). **Обязателен.**


## ⚠️ Главное правило

**При любом изменении содержимого — ВСЕГДА обновляй и TOC в корневом `README.md`, и `README.md` внутри затронутой папки.** Это касается не только добавления/удаления/переименования сервиса, ASN или страны, но и **изменения количества доменов, подсетей или Android-пакетов** в существующем `ruleset.json`. Счётчики в обоих README должны соответствовать фактическому содержимому. Вводный текст в начале корневого README менять нельзя (единственное допустимое исключение — имя файла `config.json`, если когда-либо потребуется его переименование).

После любого такого изменения:
1. Обнови `README.md` в самой папке (по единому шаблону, со счётчиками доменов/подсетей/Android-пакетов и строкой доступности).
2. Обнови таблицы (TOC) в корневом `README.md`.
3. Приведи `config.json` в соответствие (создай/обнови, либо удали, если стал ненужным — см. ниже).
4. Проверь валидность всех JSON.

## ruleset.json

- `version: 2`.
- В одном rule-объекте `domain` + `domain_suffix` объединяются по ИЛИ. `ip_cidr` и `package_name` нельзя смешивать с доменным матчером — выноси их в **отдельные** rule-объекты (они объединяются по ИЛИ между объектами).
- Домены с `*` (wildcard) → `domain_suffix`; точные → `domain`.
- ASN-папки содержат только `ip_cidr`; домены провайдера живут в `services/<name>` (домены — в services, подсети — в asn).

## config.json

Файл-конфиг папки. Содержит два опциональных ключа: `availability` и `probe`.

**Правило существования файла.** `config.json` создаётся только если есть ключ `probe` или непустой ключ `availability`; в противном случае файл не должен существовать. Бесполезных пустых файлов быть не должно.

### `availability` (разреженный)

Описывает, где сервис/ASN доступен. Базовые значения (дефолт): `blocked: false`, `geo_restricted: false`, `accredited_hosting: true`. Пишем **только отклонения** от дефолта. Если всё по дефолту — ключа `availability` нет (раньше в таких случаях был `{}`).

**`default` — только гео-ограничения сервиса.**
- `geo_restricted: true` — сервис сам ограничивает доступ по геолокации (заблокирован для региона); `false` — не ограничивает.
- Если `geo_restricted: false` — указывать его в `default` не нужно (это и есть дефолт). Секцию `default` пишем, только когда гео-ограничение по умолчанию `true`.
- Дефолтные гео-ограничения из `default` применяются к **каждой** секции страны.

**Секции стран (`ru`/`ua`/`kz`/`by`).** В каждой описываем: заблокирован ли сервис страной (`blocked: true|false`) и есть ли гео-ограничение (`geo_restricted`).
- `geo_restricted` в секции страны пишем, **только если** его статус отличается от `default` (а если `default` нет — отличается от `false`). Совпадает — не пишем.
- `blocked` — всегда про конкретную страну (страна/оператор заблокировали сервис или ASN); в `default` его не бывает.

**`accredited_hosting` (только Россия).** Если страна — Россия и сервис заблокирован страной или ASN (`ru.blocked: true`), но при этом доступен через серверы аккредитованных Минцифрой IT-компаний (Selectel, Yandex Cloud, VK Cloud) — ставим `accredited_hosting: true` (как у YouTube). Флаг ставится **только** когда `ru.blocked: true`.

### `probe`

URL для health-check (urltest) в sing-box. Требования:
- Только HTTPS, без авторизации.
- Хост URL обязан присутствовать в `ruleset.json` сервиса (`domain` или `domain_suffix`).
- Приоритет выбора URL: сначала `/generate_204` или `/status`, затем `/favicon.ico` или `/robots.txt`, затем apex-домен (если отдаёт 2xx/3xx).
- Если нет ни одного подходящего URL — ключ `probe` не добавляем.
- `probe` используется **только** в `services/`; в `asn/` и `countries/` его не бывает.
- `probe` не влияет на счётчики в README и на строку доступности.

## Данные

- Сервисы РФ (yandex, vk, госуслуги, маркетплейсы…) обычно гео-ограничены за рубежом и заблокированы в UA.
- Доступность и Android-пакеты сверяй по актуальным источникам/новостям.
